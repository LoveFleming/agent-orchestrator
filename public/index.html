🎯 什麼是 A2A？

Agent2Agent Protocol 是 Google 主導的開放標準，讓不同框架、不同廠商的 AI Agent 能夠互相發現、溝通、協作。不論你用 LangChain、CrewAI、OpenAI Agents 還是自建系統，A2A 提供統一的語言。

🔑 核心概念

Agent Card
JSON 元資料，宣告 Agent 的身份、技能、端點、認證方式
Task
最小工作單元，有狀態生命週期 (submitted→working→completed)
Message
一輪對話，含 role (user/agent) 和多個 Part
Part
內容最小單元：文字、檔案、結構化資料
Artifact
Agent 產出物（文件、圖片、資料），由 Part 組成
Context
邏輯分組，把相關的 Task 和 Message 串在一起

⚙️ 六大操作

message/send發送訊息，同步取回結果
message/stream發送訊息，SSE 串流即時更新
tasks/get查詢 Task 狀態與 Artifacts
tasks/list列出所有 Tasks
tasks/cancel取消進行中的 Task
GET /.well-known/agent.json取得 Agent Card（發現機制）

🔄 Task 狀態生命週期

submitted → working → input-required → completed | failed | canceled

input-required = 需要使用者補充資訊（Human-in-the-loop）

📡 通訊模式

⚡ Sync
HTTP POST，等 HelpDesk 處理完回傳。
適合短問答、即時查詢。
🌊 Stream
SSE 串流，即時看到處理進度。
適合長任務、生成內容。
🔔 Webhook
非同步 push notification。
適合超長任務、斷線續傳。

🏗️ 三層架構

📐 Data Model — Task, Message, AgentCard, Part, Artifact（協議無關的資料定義）
⚙️ Operations — Send, Stream, Get, List, Cancel（抽象操作）
🔌 Bindings — JSON-RPC 2.0, gRPC, HTTP/REST（具體協定綁定）
